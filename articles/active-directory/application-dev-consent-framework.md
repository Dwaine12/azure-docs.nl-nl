# <a name="how-application-consent-works"></a>Hoe werkt in toepassing toestemming geven

Dit artikel is bedoeld om u meer informatie over de werking van het Azure AD toestemming framework zodat u toepassingen effectiever ontwikkelen kunt te helpen.

## <a name="recommended-documents"></a>Aanbevolen documenten

- Ophalen van een algemeen begrip van [hoe toestemming een resource-eigenaar voor de toegang tot bronnen van een toepassing kunt](https://docs.microsoft.com/azure/active-directory/develop/active-directory-dev-glossary#consent).
- Een stapsgewijs overzicht van [hoe de Azure AD toestemming framework toestemming implementeert](https://docs.microsoft.com/azure/active-directory/develop/active-directory-integrating-applications#overview-of-the-consent-framework).
- Voor meer diepte meer [hoe het framework toestemming kunt gebruiken in een toepassing met meerdere tenants](https://docs.microsoft.com/azure/active-directory/develop/active-directory-devhowto-multi-tenant-overview#understanding-user-and-admin-consent) te implementeren 'gebruiker' en 'admin' toestemming, ondersteuning van meer geavanceerde toepassing met meerdere lagen patronen.
- Voor meer diepte meer [hoe toestemming tijdens de grant-autorisatiecodestroom in de protocollaag van OAuth 2.0 wordt ondersteund.](https://docs.microsoft.com/azure/active-directory/develop/active-directory-protocols-oauth-code#request-an-authorization-code)
